# NovaTrail 初始化设计

## 目标

将 Firefly 默认示例站点初始化为个人博客 **NovaTrail**，发布域名为 `https://novatrailx.qzz.io/`。站点不再显示原作者的个人资料、社群、邮件、支付信息、音乐、动态、相册、追番或番组数据；保留主题框架、开源许可证和必要归属。

## 已确认的内容

- 站点名称：`NovaTrail`
- 站点简介：`记录技术、思考与沿途风景。`
- GitHub：`https://github.com/Q11071`
- 背景图源文件：`D:\Blog\202408114ea7lj.webp`
- 文章发布日期与更新日期统一为 `2026-07-23`，分类统一为“博客指南”。
- 保留“友链”入口，清空预置友链；不开放留言板或评论服务。
- 保留“我的”中的动态、相册、追番、番组计划入口，删除示例数据并显示友好提示。

## 设计决策

采用“干净初始化”而非仅修改配置：所有访客可见的默认身份、数据和外链都会被替换或清空；配置页、内容页与空状态保持一致。不会重写 Firefly 的架构，也不会删除主题源码、许可证或未被站点引用的主题资源。

### 品牌与页面外观

- `siteConfig` 使用 NovaTrail 名称、站点 URL、中文描述与深蓝青色主题。
- 将用户提供的 WebP 图片复制为项目的自有图片资源，作为桌面和移动横幅；背景视频、随机二次元壁纸与视频控制入口关闭。
- 导航 Logo 与侧栏头像改为无人物的 `N` 字母标识；侧栏资料使用已确认的名称和简介。
- 公告改为 NovaTrail 欢迎语；原作者的头像、流萤形象和文本不会出现在公开页面。

### 导航、社交与关于

- “社交”仅展示友链；留言板从导航中移除，评论类型继续为 `none`。
- “链接”只保留 GitHub 主页；删除 QQ 群、Gitee、原作者文档与原作者其它外链。
- 侧栏社交链接仅保留 GitHub 和 RSS，移除原 QQ 与邮箱。
- “关于”改写为 NovaTrail 的介绍、域名与 GitHub 链接；删除原作者资料、演示站、个人站和推广内容。
- 打赏入口与原收款方式从公开导航移除，打赏数据关闭。

### 内容与站点资料

- 所有当前文章的 Frontmatter 日期统一为 `2026-07-23`，分类统一为“博客指南”。
- 默认演示文章保留为博客写作、Markdown、图表、布局等指南，但正文中的原作者个人站、社群、仓库卡片、推广链接与示例身份会被删除或替换为中性说明。
- 当前草稿示例调整为公开的写作指南，以免保留未初始化的默认草稿。
- 友链配置清空；友链页面显示“整理中”的说明，且不引用作者邮箱或评论区申请方式。

### “我的”空状态

- 删除 `src/content/dynamic/` 中的示例动态，动态页显示“尚未发布动态，欢迎稍后再来”。
- 清空相册配置与公开示例相册资源，页面显示“相册正在整理中”。
- 清除 Bangumi 用户 ID、Bilibili UID 和追番相关来源，页面显示“尚未开始记录”提示，不请求原作者的第三方数据。
- 关闭音乐播放器并清除原作者歌单；禁用侧栏最新动态的旧数据展示。
- 对主题现有的空白状态不足之处，增加一个小型可复用空状态组件；不改动页面路由或主题的其它模块。

## Cloudflare 与发布边界

- 不更改 Cloudflare 项目名、账号、DNS 或自定义域绑定；这些属于现有远端部署配置。
- `wrangler.toml` 当前已有用户未提交修改，实施时不覆盖、不纳入本任务提交。
- 推送后依赖既有 Git 集成触发 Cloudflare 构建；如没有现成集成，构建验证完成后报告该前置条件，而不擅自执行远端部署。

## 验收标准

1. 构建后的站点在标题、SEO URL、导航、侧栏、公告、友链与关于页显示 NovaTrail 信息。
2. 首页背景使用用户提供的 WebP 图，不加载默认二次元背景或远程背景视频。
3. 导航不含留言、QQ 群、Gitee、原作者文档、打赏入口；GitHub 指向 `Q11071`。
4. 所有文章日期为 `2026-07-23`，分类为“博客指南”。
5. 动态、相册、追番、番组计划入口可访问，没有原作者数据，并展示清晰空状态。
6. `pnpm check`、`pnpm type-check`、`pnpm build` 成功；构建结果不含旧站点域名、原作者个人邮箱、QQ 群、Gitee 或原作者公开个人资料。

## 风险与回退

- 用户提供的背景为横向图片；移动端使用同一资源并以 CSS 裁切显示，不生成额外图片。
- 文章主体中需要保留的开源主题说明会保留中性、可追溯的归属；不保留作者个人推广信息。
- 所有清空操作可从 Git 历史恢复。实施前会再次检查目标文件，避免删除主题运行时所需文件。
